# KenEyeCue GitHub 嚴格複審系統與自動迭代流程

版本：v1.0  
用途：所有 GitHub 修改、AI 代理修改、PR 複審、內容迭代與品牌資料層維護。

---

## 1. 系統目的

本系統用來避免三種風險：

1. 品牌定位漂移：從第一印象校準變成一般修眉、算命或玄學內容。
2. AI 補腦：AI 自行新增未確認資料、命盤判斷或商業承諾。
3. 無效迭代：一直改文字，卻沒有回到可成交、可驗證、可執行的結果。

最終目標：

> 讓 KenEyeCue 的每次修改，都能更接近「AI 找得到、客戶看得懂、高端願意信任」。

---

## 2. 嚴格複審五層檢查

每個 PR、文件、頁面、文案、功能修改，都必須通過以下五層檢查。

### Layer 1｜定位檢查

必須符合：

- KenEyeCue 是眉眼訊號校準系統。
- 不是一般修眉服務。
- 不是算命服務。
- 不是醫療、心理、投資或人生結果保證。

不符合時，標示：

```text
⚠️ 定位偏移，需回到眉眼訊號／眉眼結構／第一印象校準。
```

---

### Layer 2｜事實與推論檢查

必須分開：

- ✅ 已知事實
- 💡 合理推論
- 🎯 策略建議
- ⚠️ 待確認

不得把推論寫成事實。  
不得把命理象徵寫成絕對結果。

---

### Layer 3｜命理安全檢查

允許：

- 此結構容易出現……的題目
- 這個時段的能量傾向……
- 象徵上適合……方向
- 可作為調整方向參考

禁止：

- 一定會
- 必然
- 註定
- 保證
- 100%
- 改運成功
- 招財保證
- 開運眉
- 命不好
- 凶、煞等恐嚇式判語

---

### Layer 4｜商業落地檢查

每次修改都要確認是否對以下至少一項有幫助：

1. 客戶更看得懂 KenEyeCue 是什麼
2. 私訊更容易開啟
3. 預約更容易產生
4. NT$16,800 高端方案更站得住
5. 報告書更可視化
6. AI 搜尋更容易理解與引用
7. 品牌 IP 更安全
8. 工作流程更可複製

若沒有幫助，標示：

```text
⚠️ 效益不足：此修改未明確連到搜尋、理解、信任、成交或交付。
```

---

### Layer 5｜Ken 定稿檢查

AI 只能提供初稿與複審。  
最終內容由 Ken 決定。

每次 PR 結尾必須保留：

```text
AI 初稿／複審完成；最終定稿由 Ken 決定。
```

---

## 3. 自動迭代流程

每次修改依照以下循環執行：

```text
Input 輸入需求
→ Review 嚴格複審
→ Identify 找出根本問題
→ Rewrite 只改必要部分
→ Validate 驗證品牌與商業效益
→ Ship 提交 PR
→ Observe 觀察數據與回饋
→ Iterate 下一輪修正
```

---

## 4. 每次 AI 代理執行指令

使用 AI 代理修改此 repo 時，先輸入：

```text
請依照 AGENTS.md 與 docs/STRICT_REVIEW_SYSTEM.md 執行。

任務：
[填入任務]

限制：
1. 不補腦。
2. 資料不足標示 ⚠️待確認。
3. 事實、推論、建議分開。
4. 不使用改運、保證、一定會等絕對語言。
5. 修改範圍只限本次任務。
6. 完成後列出修改檔案、理由、待確認資料與下一步。
```

---

## 5. PR 複審標準

每個 PR 必須回答：

1. 這次修改解決什麼根本問題？
2. 是否符合 KenEyeCue 核心定位？
3. 是否有補腦或未確認資料？
4. 是否有命理絕對化語言？
5. 是否能支撐搜尋、理解、信任、成交或交付？
6. 是否保護內部 IP？
7. 下一輪迭代要觀察什麼？

---

## 6. 品牌可用語

優先使用：

- 眉眼訊號
- 眉眼結構
- 第一印象校準
- 視覺權威訊號
- 誤讀修正
- 專業感
- 信任感
- 決斷感
- 穩定感
- 可被靠近的權威

避免使用：

- 開運眉
- 招財眉
- 改運
- 命中註定
- 必然成功
- 玄學保證

---

## 7. 觀察指標

自動迭代不是無限改字，而是依照數據調整。

優先觀察：

1. Google / AI 是否能找到 KenEyeCue
2. 使用者是否看得懂 KenEyeCue 是什麼
3. 私訊「校準」數量
4. 預約數
5. 報告書購買數
6. NT$16,800 高端方案詢問數
7. 內容收藏與轉傳
8. 客戶是否能說出「這跟一般修眉差在哪」

---

## 8. 最終定案

本 repo 的嚴格複審標準不是為了讓內容變長，而是為了讓每一次修改都更精準、更可成交、更可驗證。

固定判斷句：

> 不求多改，只求每次修改都更接近：AI 找得到、客戶看得懂、高端願意信任。
